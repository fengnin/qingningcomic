# 漫画气泡布局规则表

## 目标

把对白气泡的定位从“固定左右排版”升级为“候选位置 + 评分选择”，让气泡优先落在留白区，减少遮挡脸、手、书本和关键道具的情况。

## 设计原则

1. 先避让主体，再谈美观。
2. 阅读顺序优先于角色站位。
3. 长对白必须拆短，不允许单个气泡无限变宽。
4. 对白、旁白、思考分层处理，不共用一套样式。
5. 气泡尾巴服务于说话者，但不能牺牲版面安全性。

## 气泡类型

| 类型 | 用途 | 形状 | 备注 |
|---|---|---|---|
| speech | 角色对白 | 椭圆/圆角矩形，带尾巴 | 默认类型 |
| thought | 内心独白 | 云状/弱化边框，带虚尾 | 文字应更短 |
| narration | 旁白/说明 | 矩形/圆角矩形，无尾巴 | 用于时间、地点、解释 |
| shout | 大声喊叫 | 不规则/更尖锐的气泡 | 可复用 speech 逻辑，但样式更强烈 |

## 布局流程

### Step 1: 预处理文本

| 规则 | 说明 |
|---|---|
| 文本长度限制 | 单个气泡尽量控制在 1 到 2 行 |
| 超长拆分 | 超过阈值的对白优先拆成多个短句 |
| 旁白分离 | 叙述性内容不要塞进对白气泡 |

### Step 2: 生成候选位置

默认候选区：
- 左上
- 右上
- 左下
- 右下

可选扩展候选区：
- 上中
- 左中
- 右中

候选区生成原则：
- 优先选择画面边缘和留白区
- 特写镜头允许更靠中，但仍要避开脸部
- 多人场景优先选择离当前说话者最近的安全区

### Step 3: 对候选打分

分数越低越好。

| 项目 | 权重 | 扣分理由 |
|---|---:|---|
| 覆盖脸部 | 高 | 直接影响视觉主体 |
| 覆盖手部 | 高 | 容易破坏动作表达 |
| 覆盖书本/道具 | 高 | 破坏剧情关键点 |
| 覆盖标题/文字 | 高 | 干扰阅读 |
| 与其他气泡重叠 | 高 | 造成版面混乱 |
| 靠近画面中心 | 中 | 容易抢主体 |
| 与说话者过远 | 中 | 尾巴过长不自然 |
| 贴近边缘过度 | 低 | 可能显得局促 |

### Step 4: 选择最低分候选

优先级顺序：
1. 不遮挡主体
2. 阅读顺序自然
3. 尾巴方向合理
4. 版面平衡

### Step 5: 绘制尾巴

| 规则 | 说明 |
|---|---|
| 先定框，再定尾巴 | 不要让尾巴反过来决定框的位置 |
| 尾巴尽量短 | 减少跨越关键区域 |
| 尾巴指向嘴部附近 | 但不要过分精确到压脸 |
| 尾巴可轻微偏移 | 安全性优先 |

## 评分建议

可以用简单的启发式分数，不必一开始就上复杂视觉检测。

| 维度 | 建议 |
|---|---|
| 人脸区域 | 预估角色头部位置，气泡框与其重叠直接重罚 |
| 手部区域 | 角色持物/递物时，手部区域加重罚分 |
| 道具区域 | 书、卡片、门牌、标题等关键对象加重罚分 |
| 留白区域 | 优先加分 |
| 边缘安全区 | 适度加分 |

## 长对白拆分规则

| 规则 | 说明 |
|---|---|
| 12-16 个中文字符作为参考阈值 | 仅作为布局参考，不是硬性字数限制 |
| 超过两行优先拆分 | 一句拆成两句短对白更像漫画 |
| 连续信息拆成多气泡 | 不要塞进一个超宽气泡 |
| 旁白不要伪装成对白 | 说明性内容单独走 narration |

## 推荐实现方式

### 轻量版

- 固定生成 4 个基础候选区
- 用启发式规则打分
- 选择最低分位置
- 尾巴后绘制

适合快速上线，改动小。

### 标准版

- 基础候选区 + 扩展候选区
- 根据 `shotType` 和 `character` 数量动态调整
- 引入更强的避让规则

适合持续迭代，效果更稳。

## 与现有实现的关系

当前实现已经属于“客户端后叠加气泡”，方向是对的。下一步应重点优化：
- 气泡落点选择
- 长对白拆分
- 旁白与对白分层
- 尾巴和框的解耦

不建议再继续依赖 prompt 去解决排版问题，因为那会让结果不稳定，且难以精确控制。

## 结论

漫画气泡布局的本质不是“把框放进去”，而是“在不破坏画面的前提下，把信息放到读者最容易读的位置”。最实用的做法是：候选位置 + 评分选择 + 长对白拆分 + 分层渲染。
