# 对白气泡与旁白框规范

## 目标

统一漫画项目的图内文本表达方式，避免气泡内容过长、层级混乱、遮挡主体，保证阅读顺序和漫画感。

## 核心原则

1. 对白继续使用气泡，保留漫画表达习惯。
2. 气泡只负责“说话”，不承担大段叙述。
3. 旁白、时间、地点、动作说明使用旁白框，不混入对白气泡。
4. 图内文本尽量短，长文本优先拆分或外置。
5. 气泡优先放在留白区，不遮挡脸、手、书页和关键道具。

## 文本类型定义

### 1. 对白气泡

用于角色直接说话。

特征：
- 椭圆或圆角气泡
- 带尾巴，尾巴指向说话者
- 字数短，通常 1 到 2 行

适用内容：
- 对话
- 反问
- 简短感叹

### 2. 旁白框

用于叙述、转场、时间地点说明、剧情解释。

特征：
- 矩形或圆角矩形
- 通常无尾巴
- 放在画面边缘、角落、留白区

适用内容：
- 时间信息
- 场景说明
- 叙述性文字
- 动作补充说明

### 3. 思考气泡

用于角色内心独白或思考。

特征：
- 云状、虚线或弱化边框
- 尽量短

适用内容：
- 内心活动
- 犹豫
- 自我对话

## 排版规则

1. 单个对白气泡尽量控制在 1 到 2 行。
2. 超过两行的内容优先拆句，不要硬塞进一个气泡。
3. 同一面板里气泡数量尽量少，优先保留最关键的对白。
4. 气泡优先放在空白区域，避免覆盖主体脸部、手部、书本、关键道具。
5. 阅读顺序应自然，默认从左上向右下，或者按镜头焦点顺序排列。
6. 同一章节内气泡样式保持一致，不同文本类型不要混用同一种视觉样式。

## Prompt 约束建议

在生成 Prompt 时，建议加入以下约束：

- 不要在图像中绘制大段中文正文。
- 如果需要文字，只生成少量短对白气泡。
- 旁白和说明文字不要直接画进角色对白气泡中。
- 气泡放在留白区，不要遮挡人物脸、手、书本和关键道具。
- 一张图最多保留少量文本块，避免信息过载。

## 前端渲染建议

1. 对白气泡和旁白框分字段存储，不要混成一段字符串。
2. 渲染时根据文本类型选择不同样式。
3. 先计算安全区域，再决定气泡位置。
4. 如果文本过长，优先拆分成多段或移动到图外正文区。
5. 对话与说明要有明显层级差异，避免读者误判信息类型。

## 当前项目的建议落点

1. `PromptBuilder` 或同类提示词构建模块增加对白、旁白、思考三类文本的区分规则。
2. 面板数据模型中把对白和说明字段拆开。
3. 图像生成阶段只负责画面内容，不把长文本直接画入图中。
4. UI 或渲染阶段统一处理气泡位置、字号和边距。

## 结论

漫画项目的图内文本应以对白气泡为主，但对白气泡必须保持短、清晰、位置合理。旁白和说明应独立为旁白框，不要和对白混用。这样既保留漫画感，也能显著提升可读性和稳定性。
