
# 青柠漫画 - C++ 版本

> AI 驱动的小说转漫画平台 - C++11 + Qt 5.15 实现

## 项目简介

这是原 QingningComic 项目的 C++ 复刻版本，保持原仓库的核心架构、功能逻辑和实现思路不变，仅将原实现语言从 Node.js 转换为 C++。

### 项目定位

| 维度 | 说明 |
|------|------|
| **产品类型** | 生产级 SaaS 产品 |
| **目标用户** | 漫画创作者、小说作者、内容创作者 (大规模用户，面向公众开放) |
| **部署方式** | 桌面版 + Web 版双版本 |
| **后端架构** | 完整复刻原仓库架构 (Cloudflare Workers + Queues + Supabase) |

### 目标用户群体

#### 🎨 漫画创作者

**核心痛点**：
- ⏱️ **创作周期过长**：传统手绘漫画从草稿到完稿可能需要数天甚至数周
- 💰 **试错成本高**：投入大量时间绘制后才发现剧情或构图不理想
- 🎭 **难以快速迭代**：想要尝试不同的分镜方案、角色造型或场景设计

**我们的解决方案**：
- ⚡ **快速验证**：创作者只需写好构思的剧情文本，在 **5-8 分钟内**即可转化为完整的漫画分镜预览
- 🎛️ **多层次控制**：提供详细的编辑功能，允许创作者通过**自然语言指令**精细控制生成效果
- 🖼️ **基于自有素材**：支持上传创作者自己绘制的人设图作为参考
- 🔄 **快速迭代**：对不满意的面板可以单独重新生成

#### 📚 小说读者/作者

**核心痛点**：
- 📖 **世界观呈现单薄**：纯文字小说缺乏直观的视觉体验
- 🎭 **人物形象难以统一**：读者对角色的想象千差万别
- 💝 **粉丝向内容匮乏**：喜欢某部小说的读者想要创作同人内容，但缺乏绘画技能

**我们的解决方案**：
- 🌍 **世界观可视化**：小说作者可以为自己的作品生成配套的漫画版章节
- 🎨 **人设图延伸**：基于人设图生成完整的漫画场景
- 📈 **提升读者粘性**：为小说提供配套的视觉内容，增强读者的代入感
- 💖 **同人创作神器**：粉丝可以撰写同人剧情，生成属于自己的独家同人漫画

### 核心功能

- 小说管理 (上传、编辑、删除)
- AI 文本分析 (Qwen API)
- 分镜脚本生成
- 角色圣经管理
- 面板图片生成 (Imagen API)
- 图片编辑 (inpaint/outpaint/bg_swap)
- 改稿功能 (自然语言改稿)
- 导出功能 (PDF/Webtoon/Resources)

## 技术栈

- **编程语言**: C++11 标准
- **GUI 框架**: Qt 5.15.2 (LTS)
- **编译套件**: Desktop_Qt_5_15_2_MinGW_64_bit
- **构建系统**: qmake (Qt 5.15.2)

### 2.6 消息队列详解 (Cloudflare Queues)

#### 什么是消息队列？

消息队列是一个"排队系统"，类似于银行取号或餐厅点餐：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         生活中的"消息队列"                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  银行排队取号:                                                               │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  没有排队系统: 100人同时挤在柜台前 → 柜员忙不过来 → 一片混乱          │   │
│  │  有排队系统: 取号 → 坐着等 → 按顺序叫号 → 有序高效                   │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  餐厅点餐:                                                                   │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  没有订单系统: 顾客直接冲进厨房 → 厨房崩溃                            │   │
│  │  有订单系统: 下单拿小票 → 订单进队列 → 厨师按顺序做 → 叫号取餐       │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

#### 为什么青柠漫画需要消息队列？

| 问题 | 没有队列的后果 | 有队列的解决方案 |
|------|---------------|-----------------|
| **API 超时限制** | Cloudflare Workers 有 CPU 时间限制，AI 调用超时返回 504 错误 | 同步 API 立即返回 jobId，Worker 后台慢慢处理 |
| **并发控制** | 100 个用户同时请求 → Qwen API 限流 → 大量用户收到错误 | 请求排队处理，控制并发数，避免触发限流 |
| **成本控制** | AI API 按调用收费，大量并发产生高额账单 | 控制处理速率，避免突发费用 |
| **失败重试** | 网络波动导致失败 → 用户需要重新开始 | 自动重试机制（最多 3 次），用户无感知 |
| **任务追踪** | 用户不知道处理进度 | Job 状态管理，前端轮询显示进度 |

### 为什么需要消息队列？

由于目标用户是大规模用户，消息队列是**必须的**：

| 问题 | 说明 | 消息队列解决方案 |
|------|------|-----------------|
| **API 超时限制** | Cloudflare Workers 有 CPU 时间限制，AI API 调用可能需要几分钟 | 同步 API 立即返回 jobId，Worker 后台处理 |
| **并发控制** | 大量用户同时请求会触发 AI API 限流 | 请求排队处理，控制并发数 |
| **成本控制** | AI API 按调用收费，大量并发会产生高额账单 | 控制处理速率，避免突发费用 |
| **失败重试** | AI API 调用可能因网络波动失败 | 自动重试机制，确保任务完成 |
| **任务追踪** | 用户需要知道任务进度 | Job 状态管理，前端轮询更新 |

## 六、编译说明

### 前置要求

- Qt 5.15.2 (MinGW 64-bit)
- MinGW-w64 GCC 8.1.0+
- Qt Creator (推荐)

### 方法一：使用 Qt Creator (推荐)

1. 打开 Qt Creator
2. 菜单：文件 → 打开文件或项目
3. 选择 `QingningComic.pro` 文件
4. 配置项目：选择 "Desktop Qt 5.15.2 MinGW 64-bit"
5. **配置 API Key**：复制 `config.ini.example` 为 `config.ini`，填入您的 API Key
6. 编译项目：按 `Ctrl+B`
7. 运行项目：按 `Ctrl+R`

### 配置文件说明

项目使用 `config.ini` 配置文件管理敏感信息（API Key、数据库密码等）。

**首次使用步骤：**

1. 复制配置模板：
   ```bash
   cp config.ini.example config.ini
   ```

2. 编辑 `config.ini`，填入您的配置：
   ```ini
   [qwen]
   # 从 https://dashscope.console.aliyun.com/apiKey 获取
   apiKey = sk-your-real-api-key-here
   model = qwen-plus
   
   [database]
   host = 192.168.xxx.xxx
   port = 3306
   database = qingning_comic
   username = qingning
   password = your_password
   ```

**重要提示：**
- `config.ini` 包含敏感信息，**请勿提交到 Git**
- 项目已配置 `.gitignore` 自动忽略 `config.ini`
- `config.ini.example` 是模板文件，可以提交到 Git

### 方法二：使用命令行

```bash
cd comic
qmake QingningComic.pro
mingw32-make
```
