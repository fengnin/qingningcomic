#### 任务 2.1 详细设计：NovelService（小说服务）

##### 一、功能概述

NovelService 是小说管理的核心业务层，负责处理小说的 CRUD 操作、状态管理、文本存储等功能。

##### 二、与原仓库 API 对照

| 功能 | 原仓库 API | NovelService 方法 | 说明 |
|------|-----------|------------------|------|
| 获取作品列表 | `GET /novels` | `getNovels()` | 支持分页、状态筛选 |
| 创建作品 | `POST /novels` | `createNovel()` | 创建小说记录 |
| 获取作品详情 | `GET /novels/{id}` | `getNovelById()` | 获取单个小说 |
| 更新作品 | `PATCH /novels/{id}` | `updateNovel()` | 更新小说信息 |
| 删除作品 | `DELETE /novels/{id}` | `deleteNovel()` | 删除小说及关联数据 |
| 统计作品数量 | - | `countNovels()` | 按用户/状态统计 |

##### 三、小说状态定义

```
状态流转图：

  created ──→ analyzing ──→ analyzed ──→ generating ──→ completed
                 │                           │
                 └──────────→ error ←────────┘
```

| 状态 | 英文 | 说明 | 可转换状态 |
|------|------|------|-----------|
| 已创建 | `created` | 刚创建，等待分析 | analyzing |
| 分析中 | `analyzing` | 正在调用 AI 分析 | analyzed, error |
| 已分析 | `analyzed` | 分析完成，等待生成图片 | generating |
| 生成中 | `generating` | 正在生成图片 | completed, error |
| 已完成 | `completed` | 全部流程完成 | - |
| 出错 | `error` | 处理过程中出错 | created |

##### 四、Novel 模型类属性

| 属性 | 类型 | 说明 |
|------|------|------|
| id | QString | UUID，小说唯一标识 |
| userId | QString | 所属用户 ID |
| title | QString | 小说标题 |
| originalText | QString | 原文内容（短文本） |
| originalTextPath | QString | 文件存储路径（长文本） |
| status | NovelStatus | 小说状态 |
| storyboardId | QString | 关联的分镜 ID |
| metadata | QVariantMap | 元数据（类型、标签等） |
| createdAt | QDateTime | 创建时间 |
| updatedAt | QDateTime | 更新时间 |

##### 五、NovelService 方法列表

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `getNovels()` | userId, page, pageSize, statusFilter | NovelPageResult | 分页获取小说列表 |
| `getNovelById()` | novelId | Novel | 获取单个小说详情 |
| `createNovel()` | userId, title, text, metadata | Novel | 创建新小说 |
| `updateNovel()` | novelId, data | bool | 更新小说信息 |
| `deleteNovel()` | novelId | bool | 删除小说及关联数据 |
| `updateStatus()` | novelId, status | bool | 更新小说状态 |
| `countNovels()` | userId, status | int | 统计小说数量 |
| `saveText()` | novelId, text | bool | 保存文本内容 |
| `loadText()` | novelId | QString | 加载文本内容 |
| `exists()` | novelId | bool | 检查小说是否存在 |
| `hasPermission()` | novelId, userId | bool | 检查用户权限 |

##### 六、方法功能详解

###### 6.1 获取小说列表 (getNovels)

**功能定义**：从数据库中查询并返回用户创建的所有小说，以分页形式展示。

**UI 体现**：在"作品空间"页面以卡片网格形式展示所有小说。

```
┌─────────────────────────────────────────────────────────────────┐
│  📚  作品空间                                                    │
├─────────────────────────────────────────────────────────────────┤
│  [全部] [已完成 · 1] [分析中 · 1] [已创建 · 1]                    │
├─────────────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐             │
│  │ 青柠        │  │ 星空之下    │  │ 夏日回忆    │  ← 小说列表  │
│  │ [已完成]    │  │ [分析中]    │  │ [已创建]    │             │
│  └─────────────┘  └─────────────┘  └─────────────┘             │
└─────────────────────────────────────────────────────────────────┘
```

**对应原仓库 API**：`GET /novels`

###### 6.2 获取小说详情 (getNovelById)

**功能定义**：根据小说 ID 查询并返回单本小说的完整信息，包括标题、状态、原文内容、创建时间等。

**UI 体现**：点击小说卡片后，跳转到小说详情页面，展示小说的完整信息。

```
┌─────────────────────────────────────────────────────────────────┐
│  ← 返回                    青柠                                  │
├─────────────────────────────────────────────────────────────────┤
│  状态：已完成                                                    │
│  创建时间：2024-01-15 10:30:00                                   │
│  更新时间：2024-03-20 15:45:00                                   │
├─────────────────────────────────────────────────────────────────┤
│  原文内容：                                                      │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │ 从前有座山，山里有座庙...                                    ││
│  │ ...                                                          ││
│  └─────────────────────────────────────────────────────────────┘│
├─────────────────────────────────────────────────────────────────┤
│  关联角色：3 个    关联分镜：12 个                               │
│  [查看角色] [查看分镜] [开始分析] [导出漫画]                      │
└─────────────────────────────────────────────────────────────────┘
```

**对应原仓库 API**：`GET /novels/{id}`

##### 七、NovelPageResult 分页结构

| 字段 | 类型 | 说明 |
|------|------|------|

| total | int | 总记录数 |
| page | int | 当前页码 |
| pageSize | int | 每页数量 |
| totalPages | int | 总页数 |

##### 八、信号列表

| 信号 | 参数 | 说明 |
|------|------|------|
| `novelCreated()` | Novel | 小说创建成功 |
| `novelUpdated()` | novelId | 小说更新成功 |
| `novelDeleted()` | novelId | 小说删除成功 |
| `statusChanged()` | novelId, newStatus | 状态变更 |
| `errorOccurred()` | operation, message | 错误发生 |

## 七、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| Qwen API 响应慢 | 用户体验差 | 使用消息队列异步处理 |
| Imagen API 限流 | 无法生成图片 | 并发控制 + 重试机制 |
| JSON 格式错误 | 解析失败 | JSON 修复器 + Schema 约束 |
| 图片存储成本高 | 费用超支 | 使用本地存储或云存储 |
| WASM 编译问题 | 无法部署 Web 版 | 先完成桌面版，再适配 Web |
| Docker 未安装 | 无法启动 MySQL | 使用 CentOS 虚拟机安装 MySQL |

### CentOS 安装 MySQL 8.0

```bash
# 1. 下载并安装 MySQL
sudo yum install -y https://dev.mysql.com/get/mysql80-community-release-el7-7.noarch.rpm
sudo yum install -y mysql-community-server
sudo systemctl start mysqld
sudo systemctl enable mysqld

# 2. 查看临时密码
sudo grep 'temporary password' /var/log/mysqld.log

# 3. 配置数据库
mysql -u root -p
# 在 MySQL 中执行：
ALTER USER 'root'@'localhost' IDENTIFIED BY 'Qingning123!';
CREATE DATABASE qingning_comic CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'qingning'@'%' IDENTIFIED BY 'qingning123';
GRANT ALL PRIVILEGES ON qingning_comic.* TO 'qingning'@'%';
FLUSH PRIVILEGES;
EXIT;

# 4. 允许远程连接
sudo sed -i '/^\[mysqld\]$/a bind-address = 0.0.0.0' /etc/my.cnf
sudo systemctl restart mysqld

# 5. 开放防火墙
sudo firewall-cmd --permanent --add-port=3306/tcp
sudo firewall-cmd --reload

# 6. 获取 CentOS IP 地址
ip addr show
```

### VMware 端口转发配置

```
1. VMware → 虚拟机设置 → 网络适配器 → NAT 模式
2. 高级 → 端口转发 → 添加
3. 主机端口: 3306, 虚拟机端口: 3306
```
